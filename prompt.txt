Requirements:
- Use OpenClaw as the orchestrator.
- Use project-local agents.
- The repo may start with two example agents, but the orchestration pattern should support n agents.
- Use heartbeat + tmux + shell scripts + GSD only.
- Do not use cron.
- Make the setup project-local, idempotent, and easy to understand.
- Add role briefs, setup scripts, dispatch scripts, supervisor scripts, and docs.

Dispatch policy:
- Heartbeat every 5 minutes.
- Use diff-only dispatch.
- If repo diff/status has not changed for 5 minutes, dispatch one agent.
- If repo diff/status changed, do not dispatch.
- Do not let agent session timestamp noise suppress dispatch.
- Build in explicit stop conditions so the loop knows when to stop.
- Each agent run should end with one of: continue, done, blocked, or defer.
- Do not keep redispatching a lane forever just because the repo is idle.

Verification:
- Verify both agents can be dispatched manually.
- Verify the supervisor runs in tmux.
- Verify idle repo -> automatic dispatch.
- Verify the stop-condition pattern is documented and easy to apply.
- Show the commands needed to operate the system.

Use GSD discipline, preferably gsd-quick unless a full phase is clearly needed.
